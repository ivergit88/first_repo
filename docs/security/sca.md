# Композиционный анализ зависимостей (SCA)

## Что анализируется

- `package-lock.json` через локальный генератор CycloneDX SBOM (`npm run sca:sbom`).
- Файловая система проекта через Trivy в GitHub Actions.

## SBOM

Файл `docs/security/sbom.cdx.json` создан в формате CycloneDX 1.5. В проекте намеренно нет runtime npm-зависимостей: это уменьшает supply-chain attack surface и количество транзитивных CVE.

## Уязвимости и решение

При проектировании был выбран подход "zero runtime dependencies". Это устраняет типовой класс проблем, когда учебное приложение наследует CVE из Express/middleware/шаблонизаторов без реальной необходимости.

Если Trivy найдет HIGH/CRITICAL CVE в будущем, CI завершится с ошибкой. План устранения:

1. обновить уязвимую зависимость или базовый Docker image;
2. заменить пакет на встроенный модуль Node.js;
3. добавить временное исключение только при наличии обоснования и issue на исправление;
4. повторно сгенерировать SBOM и зафиксировать результат в PR.
